# AGENTS.md

## Mission générale

Ce repository centralise les projets Ethny, les automatisations, les recettes, les devis, le contenu marketing, le plan financier, les projets créatifs et les outils internes.

Ce repository sert de base de travail pour ChatGPT, Codex, VS Code et les futures automatisations internes.

---

## Règles générales pour Codex

- Ne jamais supprimer de fichiers sans justification claire.
- Toujours proposer un plan avant une grosse modification.
- Garder une architecture simple, claire et exploitable.
- Prioriser les actions qui génèrent du chiffre d’affaires, améliorent l’organisation ou réduisent la charge mentale.
- Éviter les doublons de fichiers.
- Ne pas créer de fichiers inutiles.
- Ne pas modifier les documents sensibles sans validation.
- Documenter les changements importants.

---

## Priorité business Ethny

Pour tous les projets liés à Ethny, Codex doit prioriser :

1. génération de chiffre d’affaires ;
2. amélioration de l’organisation ;
3. gestion des clients et devis ;
4. automatisation des tâches répétitives ;
5. visibilité marketing ;
6. qualité des recettes, menus et fiches techniques ;
7. structuration du CRM ;
8. développement de l’expérience client premium.

## Positionnement Ethny

Ethny est un service culinaire premium autour de :

- chef privé ;
- traiteur événementiel ;
- cuisine fusion ;
- produits frais, saisonniers et de qualité ;
- expérience personnalisée ;
- gastronomie nomade ;
- cours de cuisine ;
- événements privés ;
- table d’hôte et production culinaire.

Le ton doit rester professionnel, premium, humain, clair et commercialement utile.

---

## Méthode avant modification

Avant toute modification importante, Codex doit :

1. lire README.md ;
2. lire AGENTS.md ;
3. lire le PROJECT_BRIEF.md du dossier concerné s’il existe ;
4. identifier les fichiers impactés ;
5. proposer un plan ;
6. attendre validation si la modification est importante.

---

## Interdictions

- Ne pas exposer de clés API.
- Ne pas inventer de données financières.
- Ne pas inventer de données clients.
- Ne pas supprimer de contenu sans validation.
- Ne pas écraser un fichier existant sans expliquer pourquoi.
- Ne pas créer plusieurs fichiers qui ont le même rôle.
